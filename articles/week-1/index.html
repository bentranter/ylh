Deploying a Static Site to GitHub Pages with Wintersmith

Or, how this site was built

I figured I’d keep it simple this week, so let’s get a static site up and running on GitHub, with Node.js and Wintersmith. Anytime I use a new or different technology, I’ll do my best to explain how to install it. Let’s get started by installing Node.

Installing Node

Head on over to nodejs.org and click the ‘Install’ button. From there, just run the installer once you’ve downloaded it, and you should be good to go. To test it out, open your console. On a Mac, its just called Terminal and can be found in your applications, or by searching with spotlight, and on Windows, its called Console and can be found from the start menu, or by searching for it (I haven’t used Windows in a while, please correct me if I’m wrong).

You can test to see if Node installed correctly by running:

$ node
Note: you don’t actually type the ‘$’ symbol, its just used to inform you that you should be running a command instead of typing code.

If it works, the Node REPL will start. You can hit Control-C on any system to exit the Node REPL. Congrats! You’ve installed Node.js.

NPM

NPM stands for ‘Nice People Matter’, or ‘Node Package Manager’ (whichever you prefer). Its a package manager for JavaScript. NPM should install automatically when you install Node, but let’s make sure its there. Just run:

$ npm

As long as you don’t get a ‘command not found’ type of error, everything is good. Now, we’ll install Wintersmith using NPM.

Install Wintersmith

This is as easy as running one command:

$ npm install wintersmith -g

Where install means download a package to your computer, -g is a flag that instructs the package to be downloaded globally, so you can use that package anywhere (as opposed to only within a certain project directory), and finally wintersmith is the name of the package we want to install.

Note that installing wintersmith may take an unusually long time to install (like more than five minutes), but that’s okay, nothing’s going wrong so just be patient :)

Generate A New Site

This is as simple as running:

$ wintersmith new project-name

where project-name is obviously the name of your site. From here, enter the directory of your new site and preview it by running:

$ cd project-name
$ wintersmith preview

and open up localhost:8080 in a web browser. You’ll see a website built with the default theme.

Writing A New Post

Writing a new post is as simple as creating a new folder in the articles directory, and creating a new markdown document within the new directory. To do this, we’ll stop the server (type Control-C), and then run a few simple commands:

$ cd contents/articles
$ mkdir new-post
$ cd new-post
$ touch new-post.md

What we’ve done here is change into the directory named articles, created a new folder using the $ mkdir command, navigated to that folder, and created a new markdown document using the $ touch command. From here, you should go to a sample post to copy/paste the metadata (the stuff at the top of the document in between the ---‘s) and replace it with your own info. At this point, you can write your post! Check out this mardown cheatsheet for a handy reference on how to write in markdown if you’re new to it.
